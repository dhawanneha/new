Challenge 1

A traveler’s flight is canceled due to a severe storm at the destination. How does her airline app inform her about this and what does it say?



How would I deliver the message to the user

I would use app notification to deliver the message since users don't open their airline apps often.

However, since the information is critical and some users might not have a mobile app installed, I would also send the information via a text message and email. A text message does not require internet connection. An email is useful if there is free airport wifi but no mobile signal.

Rationale behind the text

The headline includes flight number to be specific. The body informs that bad weather in Detroit caused flight cancellation and also conveys information about the time frame of bad weather. Clicking on more information will open a dailog box in the app with more details.

Challenge 2

Critique of Beer52: Payment failed message



What are the problems with the current text?

  • The error message is too wordy. Users scan and don't read but it is not exactly easy to scan this text
  • The heading does not inform users about the problem.
  • The message also defies the law of proximity. The text and call to action button are not in proximity and there are details of orders sandwiched between them.
  • There is more text below the call to action which is again verbose and might be ignored by users.

How would I fix them?

The updated copy does the following:

  • The title informs the user about the problem
  • The error message guides the user on how to fix the problem in a concise and clear manner
  • The error message and CTA are in proximity
  • The text below CTA is concise and clear. In case, the user cannot fix the problem, then this text might be useful.

Challenge 3

Write a promotional screen for an app that lets a user choose teams, sends game reminders, real-time score updates and highlight videos.



Rationale behind the title

Fans don’t want to miss games of their favorite teams and the title promises that.

Following are the other versions I considered:

“Watch games, get scores, & see highlights”

Or

“Never miss another game”

The first choice informs the users about what the app does but does not tell the user why they might want to install it. It does not build an emotional connection. The second choice assumes that the users want to watch all the games which might not be true.

Rationale behind the body

The body conveys the value proposition in detail. It also clearly states why users might be interested in installing this app. The body first informs the user about the value of the app and then communicates what they need to do. It also informs what users need to do to get that value.

The body could be shorter without “We‘ll do the rest”.But “we’ll do the rest” conveys that the product cares for the user so, I chose to include it. I also used “pick” instead of “select” because it sounds more casual. “Select” might be less ambiguous (especially for localization) but sounds more formal. Pick is clear for English users in America. This app focuses on college teams in America so it might not need localization.

Rationale behind buttons

Users hate signing up. I removed the option to create an account upfront and let the users select the teams to get updates. Users can sign up later if they want to. Alternatively, existing users might want to sign in to retrieve all details from their account so that is the other option.
